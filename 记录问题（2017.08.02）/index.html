记不起来太多，写下一个body的特异功能，说不定后续会联系起来*

1、body的显示性：

<body>
    <p>这是默认的一行段落文字</p>
</body>

       
我们都知道，p元素默认带有上下16px的margin，但是就像上面这种情况，body的高度竟然只有一行文字的高度，不会将margin计算在内。但是，

 body {
     border: 1px solid pink;
 }

       
这样就会计算在内，同理，这个不仅是body才有的，div元素包一个p元素，也同样如此，不知什么原因，这种做法让我感受到现实的因素，说不清楚

2、video元素的特异属性：

问题背景：我在video视频上设置了一个按钮垂直水平居中的按钮，可是却点击不到，原来是被放到了video元素的下面去了，我就怀疑了，绝对定位的元素应该在文档流其它元素之上才对；还有个问题，既然放到了下面去了，但是却清晰可见。最后发现video元素浏览器代理（chrome）有个如下属性：（值得好好研究下）

object-fit: contain;

先不研究，把项目做完再来跟你纠缠不清

视频默认的长宽比：这个好像不能改变

3、background-size: x y; 几个属性值的理解：

（1）、auto

       
如果只设置了一个值，则另一个值默认auto，那auto是什么意思？就是以这一个值，根据长宽比，来计算另一个

（2）、百分比值

       
如果是以百分比为单位，则背景图片长、框是以父元素的长、框的百分比计算的，而不仅仅只是以父元素的宽度计算

       
同理，如果只设置了一个，另一个依然auto，计算的方式还是保持图片原始长宽比

（3）、background-size： cover/contain：

       
cover 和 contain的共同点：

       
同：都对背景图片scale拉伸了，并且保持图片原长宽比例不变

       
异：cover是最小比例 拉伸，使得图片覆盖“背景定位区” background position area；

               
contain 是最大比例 拉伸，使得图片“容纳于”背景定位区

（4）、所有background的缩写： background

background: url() no-repeat left top / 100%;
background-position 和 background-size 因为都是数值，容易混淆，所以之间用 `/`斜线 隔开，并且前后都要设置值

总结： 无论是图片还是视频，这个默认长宽比还是挺重要的。还有一点，任何一个很小的效果，实现起来，需要考虑的问题还是蛮多的，就拿这个背景图片大小来说

确定大小的div里，设置一张背景图片，图片有大有小，当图片比div大，怎么处理？当图片比div小，又当如何处理？这都是设计之初需要考虑的问题的。带着这些问题去学习，很多值的用法就一目了然了！
